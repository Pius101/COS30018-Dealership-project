🚗 JADE RMA — Manual Agent Walkthrough

How to add Dealer + Buyer agents from the JADE GUI and run a complete negotiation

1
Launch the HOST (with JADE RMA)
Launcher Dialog
▶
Run
Start the application — negotiation.Launcher in IntelliJ, or double-click the fat JAR
🔘
Select role
Choose HOST — start the platform
☑
Important — check this box
Tick "Show JADE RMA GUI" — this opens the agent monitor window alongside the Broker GUI
🚀
Launch
Click LAUNCH
✓ Two windows open: the Broker GUI (purple header) and the JADE RMA (JADE's own agent monitor). The RMA tree shows Main-Container containing ams, df, rma, and broker.
2
Understand the JADE RMA layout
JADE RMA Window
Agent Platforms
└── 192.168.x.x:1099/JADE
    └── Main-Container ← right-click here to add agents
        ├── ams     (Agent Management System)
        ├── df      (Directory Facilitator — yellow pages)
        ├── rma     (this GUI itself)
        └── broker  (our BrokerAgent — registered in DF)
💡
You can double-click broker to inspect its AID and current state. The Broker GUI is separate — the RMA just shows the JADE-level view of the agent.
📡
The Broker GUI has a 📡 Sniffer button. Click it now — a Sniffer window opens. You can drag broker into the sniff area later to capture all messages as a sequence diagram.
3
Add a Dealer Agent from the RMA
JADE RMA Window
🖱
Right-click
In the RMA tree, right-click Main-Container → select "Start New Agent"
✏
Fill in the dialog
Agent Name: Dealer1
Class Name: negotiation.agents.DealerAgent
⚠
JADE quirk: After typing in the Class Name field, click into the Agent Name field (or any other field) before clicking OK. The OK button only activates after you leave the text field.
✓
Confirm
Click OK
✓ Dealer1 appears under Main-Container in the RMA tree. The Dealer GUI window opens automatically — it finds the broker via the DF and is ready for use.
[10:23:52] Dealer Agent 'Dealer1' started. AID: Dealer1@192.168.x.x:1099/JADE
[10:23:52] Broker found in DF: broker@192.168.x.x:1099/JADE
[10:23:52] Connected to Broker: broker@192.168.x.x:1099/JADE
4
Add a Buyer Agent from the RMA
JADE RMA Window
🖱
Right-click again
Right-click Main-Container → "Start New Agent"
✏
Fill in the dialog
Agent Name: Buyer1
Class Name: negotiation.agents.BuyerAgent
⚠
Same JADE quirk — click into another field after typing Class Name, then click OK.
✓ Buyer1 appears in the RMA tree. The Buyer GUI window opens. RMA tree now shows: ams · df · rma · broker · Dealer1 · Buyer1
[10:24:01] Buyer Agent 'Buyer1' started. AID: Buyer1@192.168.x.x:1099/JADE
[10:24:01] Broker found in DF: broker@192.168.x.x:1099/JADE
[10:24:01] Connected to Broker: broker@192.168.x.x:1099/JADE
💡 At this point you have 3 custom agents running on the same JADE platform, all communicating via ACL messages. You can add more — repeat for Dealer2, Buyer2, etc.

7
Broker manually creates the assignment
Broker GUI Window
📑
Navigate
In the Broker GUI, click the 🔗 Assign tab
🖱
Select listing (top table)
Click the row showing: A3F9B2C1 · Dealer1 · Toyota · Camry · 2022 · 85000 · Used
🖱
Select buyer (bottom table)
Click the row showing: REQ001 · Buyer1 · Toyota · Any · 2020 · — · 80000 · Any
📝
Optional broker note
Type in the note box: Good match — car is within mileage limit, price is negotiable
🔗
Assign
Click 🔗 Assign Selected → confirm the dialog → YES
✓ Both agents are notified simultaneously. Dealer1's GUI gets a new 💬 Buyer1 tab. Buyer1's GUI gets a new 💬 Dealer1 tab. Both tabs switch into focus automatically.
[10:24:45] ★ Assignment created [NEG8F2A] Dealer: Dealer1 ↔ Buyer: Buyer1 Listing: A3F9B2C1

[10:24:45] → to Buyer1         [ASSIGNMENT_NOTIFY]         Dealer: Dealer1 Listing: A3F9B2C1

Buyer accepts — deal closes
Buyer1 GUI   Broker GUI

Buyer1 GUI — sees RM 81,000 offer
Price field shows 81000 (auto-filled from dealer's offer)
✅

  • Buyer1 GUI chat shows: ─── ✅ DEAL COMPLETE — Final price: RM 81,000 ─── — input disabled
  • Dealer1 GUI chat shows the same banner — input disabled
  • Broker GUI "💬 Negotiations" tab → completed deals table gains a new row
